# PJSK 歌词猜曲插件

## 功能简介

本插件是一款专注于《初音未来 缤纷舞台》的歌词猜曲游戏插件。用户根据歌词片段猜出歌曲名称

歌曲资源截至日服5.5周年前

## 游戏玩法

### 基本流程

1. 发送 `歌词猜曲` 开始游戏
2. 系统会随机选择一首歌曲，展示20行连续的歌词片段
3. 同时显示10个选项（包含正确答案）
4. 在限时内输入数字(1-10)选择你认为正确的答案
5. 答对获得1分，答错或超时不计分

### 双重作答机制（v1.0.1新功能）

从v1.0.1版本开始，游戏引入了**双重作答限制**机制：

#### 📊 工作原理

1. **玩家独立作答次数限制**
   - 每位玩家有独立的作答次数配额
   - 当某位玩家用完自己的配额后，该玩家无法继续作答
   - 但其他玩家仍可继续参与

2. **全局作答次数限制**
   - 整轮游戏有一个总作答次数上限
   - 所有玩家的作答次数累计计算
   - 当总次数达到上限时，无论是否有人答对，本轮自动结束

3. **协同工作规则**
   - **任一限制触发即终止回合**：
     * 如果某位玩家用完个人配额 → 该玩家被禁止继续作答
     * 如果全局次数耗尽 → 本轮立即结束
     * 如果有人答对 → 本轮立即结束（优先级最高）
   - 系统会实时提示剩余次数信息

#### 💡 使用示例

**场景1：默认配置（1次/人，5次全局）**
```
玩家A回答错误 → A剩余0次，全局剩余4次
玩家B回答错误 → B剩余0次，全局剩余3次
玩家C回答正确 → 🎉 C获得1分！本轮结束
```

**场景2：宽松配置（3次/人，10次全局）**
```
玩家A回答错误 → A剩余2次，全局剩余9次
玩家A再次回答错误 → A剩余1次，全局剩余8次
玩家B回答错误 → B剩余2次，全局剩余7次
... (继续直到有人答对或全局次数耗尽)
```

#### ⚙️ 配置建议

- **新手友好模式**：`max_attempts_per_player: 3`, `max_attempts_total: 10`
- **竞技挑战模式**：`max_attempts_per_player: 1`, `max_attempts_total: 3`
- **休闲娱乐模式**：`max_attempts_per_player: 2`, `max_attempts_total: 8`

### 歌词展示模式（v1.0.1新功能）

#### 🖼️ 图片模式（image）

- **特点**：生成精美的歌词图片，视觉效果好
- **适用场景**：追求美观体验、网络状况良好
- **输出形式**：图片文件

#### 📝 纯文本模式（text）

- **特点**：直接以文本形式发送歌词，加载速度快
- **适用场景**：网络较慢、希望快速开始游戏、节省带宽
- **输出形式**：纯文本消息

#### 🔧 切换方法

在 `_conf_schema.json` 中修改 `lyrics_display_mode` 字段：

```json
{
  "lyrics_display_mode": "image"  // 或 "text"
}
```
